# START HERE

Read this before using or editing the AI Automation Innovators operating brain.

This repo is meant to give AI tools and humans enough context to do useful work without guessing, overbuilding, or rewriting the business from scratch.

## Working Order

1. **Read `01-business/` first.**  
   Use it as the source for business identity, positioning, audience, offer language, and operating context when it exists.

2. **Read the relevant project folder second.**  
   Use the project folder to understand the current task, constraints, drafts, scope, and next action.

3. **Check proof/examples when writing persuasive material.**  
   Use proof, examples, wins, audit samples, or client language when writing content, outreach, sales language, or audit material.

4. **Check `public-ai-visibility/` for visibility work.**  
   Use it for LLM visibility, local SEO, entity files, and weekly visibility testing when that folder exists.

5. **Ask for missing context instead of guessing.**  
   If a fact, claim, client result, tool, price, scope, or folder is missing, ask for it or state the assumption clearly.

6. **Keep work scoped and small.**  
   Make the smallest useful change. Do not redesign the repo, create systems, or add tools unless specifically asked.

## Common Use Cases

### Write a Skool post

Read `01-business/`, then any relevant content, proof, or examples. Keep the post practical, direct, and outcome-focused.

### Draft a local outreach email

Use business context, the Workflow Leak Audit offer, and Central Texas positioning. Keep the email short and specific. Do not invent client details.

### Improve Workflow Leak Audit copy

Read the offer context first. Improve clarity, outcomes, objections, and next steps without changing the offer unless requested.

### Create a Central Texas Story Guide video description

Use the relevant project folder if it exists. Keep the description local, concrete, and useful. Do not invent places, people, dates, or claims.

### Review Operator Inbox OS scope

Read the project folder if it exists. Identify what is in scope, what is unclear, and what should be deferred. Avoid adding features unless requested.

### Run an AI visibility test

Use `public-ai-visibility/` if available. Record the prompt, model/tool used, date, observed result, and any follow-up needed.

### Document a win or proof point

Capture the source, date, context, result, and where it can be reused. Do not inflate the claim or remove important limitations.

## Default Rules

- Use markdown.
- Prefer plain language.
- Lead with business outcomes, not AI tools.
- Use Workflow Leak Audit as the primary offer unless told otherwise.
- Keep Belton, Texas as the canonical local base.
- Treat Central Texas as the broader service area.
- Summarize assumptions and files changed when done.
