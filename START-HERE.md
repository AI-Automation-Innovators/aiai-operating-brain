# Start Here

## Purpose

Quick-start path for humans and AI tools using this operating brain.

This file tells a new session what to read before acting so changes stay factual, small, and aligned with Kevin Tunis / AI Automation Innovators.

## Reading order

1. `README.md` — understand what the repository is and is not.
2. `AGENTS.md` — follow the repository-wide rules before editing.
3. `01-business/` — use this as the core source of truth for business context, voice, offers, and constraints.
4. Relevant project folder under `02-projects/` — read the project-specific instructions before changing project content.
5. `04-proof-and-examples/` — read only when writing proof, sales language, content, examples, objections, or other public-facing material.

## How to work safely

- Preserve the current numbered folder structure.
- Use existing markdown files as source material.
- Do not invent facts, proof, pricing, testimonials, guarantees, client names, or revenue claims.
- Mark unknown details as `TBD`.
- Keep edits practical, direct, and reviewable.
