# AGENTS.md

Rules for AI agents working in this repository.

## Core Rules

- Edit only the files requested by the user.
- Keep changes small, practical, and easy to review.
- Do not invent facts, results, clients, tools, prices, dates, or folder contents.
- Do not add tools, apps, automations, dependencies, or workflows unless specifically requested.
- Do not overbuild. Make the smallest useful improvement.
- Prefer markdown for documentation and operating notes.
- Ask clarifying questions when required facts are missing.

## Business Positioning Rules

- Preserve Kevin Tunis / AI Automation Innovators positioning.
- Lead with business outcomes, not AI tools.
- Use Workflow Leak Audit as the primary offer unless told otherwise.
- Keep Belton, Texas as the canonical local base.
- Treat Central Texas as the broader service area.
- Avoid hype, vague promises, and inflated claims.

## Repository Use Rules

- Read `START-HERE.md` before making broad changes.
- Read `01-business/` first when it exists and the task involves business context.
- Read the relevant project folder before editing project-specific work.
- Check proof/examples before writing content, outreach, sales language, or audit material.
- Check `public-ai-visibility/` for LLM visibility, local SEO, entity files, or weekly visibility testing.
- If a referenced folder or fact is missing, say so instead of guessing.

## Change Reporting

After each task, summarize:

- Files changed.
- What changed.
- Assumptions made.
- Any recommended next task.
