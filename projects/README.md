# Projects

This folder holds active build instructions, project notes, and status.

## How to add a project

Create one folder per active project only when there is real work to track. Each project should include a short `README.md` with:

- Purpose.
- Current status.
- Owner or main contact, if appropriate.
- Important constraints.
- Next actions.
- Links to relevant proof, offer, or business-context files.

## Project rules

- Keep project notes factual and current.
- Do not store speculative claims as proof.
- Archive or remove stale project instructions when they no longer help.
- Avoid adding tooling or automation unless explicitly requested.
