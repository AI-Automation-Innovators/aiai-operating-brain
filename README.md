# aiai-operating-brain

Portable operating brain for Kevin Tunis / AI Automation Innovators.

## What this repo is

This repository is a markdown-first source of truth for business context, offers, project instructions, voice guidance, proof assets, examples, and AI visibility notes.

It is designed to be easy for humans, LLMs, coding agents, Claude Code, Codex, and future automation systems to navigate without needing extra software.

## Who it is for

- Kevin Tunis and AI Automation Innovators.
- AI assistants helping with business, project, content, or operating-system work.
- Future agents that need clear source material before making changes.

## How humans should use it

- Start with `START-HERE.md` for the recommended reading path.
- Use `01-business/` as the core business context.
- Use `02-projects/` for project-specific instructions and notes.
- Use `04-proof-and-examples/` only when working with proof, examples, sales language, or public-facing content.

## How AI tools should use it

- Read `AGENTS.md` before editing.
- Treat the numbered folder model as canonical.
- Use existing markdown files as the source of truth.
- Mark unknowns as `TBD` instead of guessing.
- Keep edits small, factual, and reviewable.

## What this repo is not

- Not an app, database, package, workflow system, or build system.
- Not a place to invent proof, client names, pricing, guarantees, revenue claims, or private personal details.
- Not a replacement for confirmed business decisions from Kevin.

## Structure

- `01-business/` — core context about Kevin Tunis, AI Automation Innovators, voice, offers, and constraints.
- `02-projects/` — project-specific instructions and working notes.
- `03-public-ai-visibility/` — entity summaries, test prompts, and visibility tracking.
- `04-proof-and-examples/` — reusable proof, examples, objections, and language.
- `START-HERE.md` — quick entry point for humans and AI tools.
- `AGENTS.md` — repository-wide instructions for future AI agents.
