# aiai-operating-brain

Portable operating brain for Kevin Tunis and AI Automation Innovators.

This repository gives AI tools a shared source of context for business strategy, offers, projects, proof, voice, and AI visibility work. Use it to help Codex, ChatGPT, Claude, Gemini, and future AI tools understand the business before writing, planning, auditing, or editing.

## Repository Purpose

This repo exists to make AI-assisted work faster and more consistent without rebuilding context from scratch each time.

It should help an AI tool quickly understand:

- Who Kevin Tunis is.
- What AI Automation Innovators does.
- What offer should be treated as primary.
- Which project folder matters for the current task.
- Which proof, examples, and visibility files should support the work.
- How to update the system without overbuilding it.

## Current Repository Structure

At the time this guide was written, the repository contains only root-level operating files. The folder structure referenced below is the intended operating structure for this repo and should be created or updated only when specifically requested.

```text
aiai-operating-brain/
├── README.md
├── START-HERE.md
├── AGENTS.md
└── .gitignore
```

## Top-Level Folder Guide

Use existing folders as context when they are present. Do not invent folder contents if a folder does not exist yet.

- `01-business/` — Business identity, positioning, primary offer, audience, market, and core operating context.
- Project folders — Specific initiatives, assets, scopes, drafts, and instructions for active or recurring work.
- Proof/examples folders — Wins, examples, before/after material, client language, audit samples, or reusable proof points.
- `public-ai-visibility/` — LLM visibility, local SEO, entity files, visibility checks, and related testing notes.

If a requested folder is missing, ask for context or recommend a small next step instead of creating a broad structure.

## Primary Business Context

- Kevin Tunis is the person this operating brain supports.
- AI Automation Innovators is the business.
- Belton, Texas is the canonical local base.
- Central Texas is the broader service area.
- Workflow Leak Audit is the primary offer unless Kevin says otherwise.
- Lead with business outcomes before naming AI tools, apps, or automations.

## How Kevin Should Use This Repo With AI Tools

When starting a task with any AI tool:

1. Point the tool to `START-HERE.md` first.
2. Point the tool to `01-business/` if it exists.
3. Point the tool to the relevant project folder for the task.
4. Include proof/examples when the task involves content, outreach, sales language, audits, or claims.
5. Include `public-ai-visibility/` when the task involves LLM visibility, local SEO, entity work, or visibility testing.
6. Tell the tool the exact files it may edit.
7. Ask for small, practical updates instead of large rebuilds.

## What Should Be Updated Regularly

Update these areas when facts change or new proof is created:

- Business positioning and offer language.
- Workflow Leak Audit messaging and scope.
- Active project notes and next steps.
- Proof points, wins, examples, and useful client language.
- Public AI visibility tests and entity information.
- Local market details for Belton and Central Texas.

## What Should Not Be Changed Casually

Do not casually change:

- Kevin Tunis / AI Automation Innovators positioning.
- The primary offer from Workflow Leak Audit to something else.
- Belton, Texas as the local base.
- Central Texas as the broader service area.
- Proven examples, wins, or claims without source context.
- Folder structure or operating rules without a clear reason.

Keep this repo practical. Prefer clear markdown, small edits, and real business usefulness over elaborate documentation.
