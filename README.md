# AI Automation Innovators Operating Brain

This repository is the portable operating brain for Kevin Tunis and AI Automation Innovators. It gives human collaborators and AI agents a shared source of truth for business context, offers, active projects, proof, examples, and AI visibility assets.

It is not a software product. Do not add application code, automation, or complex systems unless Kevin explicitly asks for them.

## Where to start

1. Read [`START-HERE.md`](START-HERE.md) for the fastest orientation.
2. Read [`AGENTS.md`](AGENTS.md) before asking Codex, Claude, ChatGPT, or another agent to work in this repo.
3. Use the folder guide below to find the right context before creating or editing anything.

## Folder guide

| Path | Purpose | Update frequency |
| --- | --- | --- |
| `business-context/` | Stable positioning for Kevin Tunis, AI Automation Innovators, target market, voice, geography, and operating principles. | Update when positioning changes. |
| `offers/` | Current service and offer descriptions, especially the Workflow Leak Audit. | Update when offers change. |
| `projects/` | Instructions, status, and notes for active builds or client/internal projects. | Update during active work. |
| `proof/` | Real examples, artifacts, case-study drafts, screenshots, and results that can be supported. | Update when new proof exists. |
| `ai-visibility/` | Reusable public-facing descriptions and assets that help AI systems understand the business accurately. | Review periodically. |

## Operating principles

- Keep the repo lean and readable.
- Prefer one clear source of truth over repeated instructions.
- Do not invent clients, testimonials, dates, revenue, savings, metrics, or outcomes.
- Label drafts clearly.
- Keep Central Texas, Belton, Temple, and nearby business relevance accurate and practical.
- Preserve the positioning of the Workflow Leak Audit as a practical diagnostic offer for finding workflow gaps, bottlenecks, and automation opportunities.
- Make small, reversible edits unless a larger change is explicitly requested.

## What should not be changed casually

- Core business positioning in `business-context/`.
- Current offer language in `offers/`.
- Proof files in `proof/`; these must stay evidence-based.
- Agent safety rules in `AGENTS.md`.

## Repository status

This repo is intentionally lightweight. The current structure establishes the minimum useful operating brain without adding unnecessary tooling, code, or automation.
